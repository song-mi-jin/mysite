5. 테스트 프레임워크에 설명으로 올바른 것은?

- Junit 테스트 프레임워크는 C 기반의 오픈소스 프레임워크이다.
- 정적 분석 도구이다.
- 구현된 SW를 실행하지 않고 테스트하는 도구이다
@- 개발자의 반복적이고 시간이 많이 소요되는 테스트 작업을 자동화하여 테스트에 소요되는 시간과 노력을 절감할 수 있게 한다.

6. 다음 중 빌드 스케줄 관리도구의 기능에 대한 설명으로 올바르지 않은 것은?

- 빌드 작업 스케줄링 : 빌드 작업의 작업 주기, 작업 시간을 설정한다
- 빌드 작업 상태 및 이력 관리 : 등록된 빌드 작업의 진행 상태를 대기 중, 진행 중, 완료, 오류 유형으로 구분하여 관리하고 수행 이력을 관리한다.
@- 빌드 도구 연계 관리 : 이메일을 통해서 관련 개발자, 관리자들에게 빌드 수행 결과를 제공한다.
-  빌드 수행 결과 리포팅 : 빌드 수행 결과 오류 사항, 코드 인스펙션 점검 결과, 테스트 수행 결과 등을 웹 화면을 통해서 보여 주고, 관련자들이 공유할 수 있도록 리포팅을 제공한다

- 빌드 수행시마다 해당 빌드 버전에 해당하는 버전을 정의하는 규칙을 정의한다.

10. 애플리케이션 배포에 대한 내용으로 올바르지 않은 것은?


- 애플리케이션이 배포된 운영환경의 서버 IP, 디렉터리, 접근 계정을 확인한다.
- 개발자는 배포된 프로그램상에 오류가 발생된 경우 오류 원인을 파악하여  필요시 배포 담당자를 포함하여 관련 개발자와 협업을 통해서 오류 원인을 파악한다
 - 파일 직접 쓰기가 필요하거나 파일 전송이 필요한 경우, 인프라 관리자에게 별도의 계정을 사전에 부여받도록 한다.
@- 애플리케이션 배포 결과 오류 발생 시 배포 환경에서 즉시 수정한다.
